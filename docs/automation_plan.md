# 自动化执行方案（执行导向）

## 目标
构建每日自动化闭环：
1. 按 Zn 负极界面机制主题检索并抓取新文献。
2. 下载 PDF 并去重后入库 Zotero。
3. 生成结构化摘要与标签（默认 GPT API，Zotero 手动兜底）。
4. 同步文献笔记到 Obsidian 并建立双向链接。
5. 输出 Markdown 日报与周报。

## 管线阶段
1. 检索
- 输入：configs/research_profile.yaml
- 输出：候选文献列表（jsonl）
- 规则：采用多查询并行召回（主题词拆分），再做合并去重，避免单条检索式过窄。

2. 筛选
- 规则门控：
  - 至少命中 2 个方法学证据词
  - 过滤仅强调性能的摘要
  - 无摘要条目按主题相关性低置信保留，进入后续复核
  - 按机制相关性评分
- 输出：筛选后列表（jsonl）

3. 下载与去重
- 下载 PDF 并校验文件大小与校验和。
- 先按 DOI 去重，再按标题归一化与哈希去重。
- 输出：规范化归档与索引映射。

4. Zotero 入库
- 在目标集合中创建或更新条目。
- 挂载 PDF 与元数据。
- 按主题与机制类别打标签。

5. 摘要与笔记构建
- GPT API 路径：
  - 提取研究问题、方法、关键证据、机制解释、局限性
- 手动兜底路径：
  - 若存在 Zotero 插件笔记则保留并可覆盖
- 输出：每篇论文一份 Markdown 笔记。

6. Obsidian 同步
- 写入包含 frontmatter 的笔记：
  - citation key、DOI、期刊、年份、相关性评分、链接
- 生成主题索引页与待解决问题页。
- 自动生成日报与周报，支持固定目录增量更新。

7. 报告输出
- 日报：
  - 当日新增、重点论文、下载失败、低置信摘要
- 周报：
  - 趋势聚类、机制图谱更新、后续跟进清单

## 建议调度
- 每日任务：本地时间 07:30
- 周报任务：周日 20:00

## 验收标准
- 连续两次运行必须幂等。
- 同一 DOI 不应重复写入 Zotero。
- 每篇筛选论文都有可追溯决策记录。
- 每份生成笔记都能反链回 Zotero key。
